# 共有前セキュリティチェック

## 結論

Supabase設定後は、ログイン情報と会員状態はSupabaseで管理できます。
ただし、記事本文を本当に有料保護するには、記事本文もSupabase側へ移す必要があります。

## 今のまま共有してよい範囲

- デザイン確認
- 記事の読みやすさ確認
- 招待コード登録の流れ確認
- ログイン後に記事が見える体験確認

## 今のまま避けること

- 重要なパスワードを使うこと
- 有料記事や外に出したくない記事を載せること
- 招待コードをサイト上に公開すること
- 本番会員サイトとして課金ユーザーへ提供すること

## まだ完全な有料保護ではない理由

GitHub Pagesは静的サイトです。
画面上はログイン後に記事を表示する作りでも、記事データや画像ファイル自体は公開URL上にあります。

本当に守りたい記事は、ログイン済みユーザーだけがサーバーから本文を取得できる構成にする必要があります。

## 次の本番化候補

最短候補:

```text
GitHub Pages
Supabase Auth
Supabase Database / Storage
```

この構成にすると、メール/パスワードログイン、招待コード管理、記事本文の保護をサーバー側で扱えます。

## 入力項目

登録で集める情報は少なくします。

- ログインID/メール
- パスワード
- 招待コード

ゲスト名は不要です。
